# دليل التثبيت والتسجيل والتوزيع الأمني | كروتك (Kurotek) 🛡️

تمت ترقية هذا المشروع وتأمينه بالكامل من خلال هيكلية موزعة جديدة ومستقلة تفصل بين واجهات مستخدم نظام الأندرويد وإدارة تراخيص السيريالات المشفرة المرتبطة بمعرفات الهواتف الفريدة لمنع الاستنساخ.

---

## 📂 خريطة المشروع والملفات المضافة

1. **لوحة التحكم الخارجية (`/admin_panel.html`)**: لوحة إدارة ترخيص ويب تفاعلية مستقلة وخارجية، تضمن عدم وجود أي كود للمشرفين داخل تطبيق الأندرويد لصد هجمات الهندسة العكسية.
2. **سيرفر الـ REST API للمصادقة والتراخيص (`/server/api_server.js`)**: خادم Node.js معزز بأمان JWT، وتحقق من توقيع الطلبات عبر تشفير HMAC-SHA256، وتثبيط هجمات التكرار (Replay Attacks)، وتأمين مدخلات قواعد البيانات ضد هجمات SQL Injection.
3. **هيكلية قاعدة البيانات (`/server/database.sql`)**: مخطط SQL جاهز للاستيراد في بيئات SQLite أو MySQL لإدارة السيريالات والعملاء وسجلات الأمان والمراقبة.
4. **كود التحقق الأمني بالجهاز (`/app/src/main/java/com/example/security/DeviceSecurity.kt`)**: محرك برميوم يكشف كسر الحماية (Root Detection) وبيئات المحاكاة (Emulator Detection) ويولد معرّف هاردوير آمن (Secure Device ID).
5. **الربط مع السيرفر والتحقق المستقل (`/app/src/main/java/com/example/security/SecurityApiService.kt`)**: عميل برميوم يرسل طلبات مشفّرة وموقعة بتشفير هاردوير آمن مع ميزة السقوط الآمن للوضع الذاتي غير المتصل (Offline Fallback).
6. **قواعد تشفير الأكواد (`/app/proguard-rules.pro`)**: قواعد Obfuscation و Shrinking قوية عبر R8/ProGuard تصعّب استخراج ملفات الأكواد وسيريالات الاتصال للقرصنة.

## 🌐 أولاً: دليل تشغيل وتثبيت لوحة التحكم والـ API (على استضافة أو VPS)

### 1. إعداد وتثبيت قاعدة البيانات (`database.sql`)
- في حال استخدام **MySQL / MariaDB**: قم بإنشاء قاعدة بيانات جديدة وقم باستيراد الجداول من الملف `/server/database.sql`. (تأكد من استبدال `AUTOINCREMENT` بـ `AUTO_INCREMENT` في حال الترحيل لـ MySQL).
- في حال استخدام **SQLite**: يقوم خادم Node.js بالربط مباشرةً تلقائياً لسهولة التوزيع.

### 2. تشغيل سيرفر الـ API (`api_server.js`)
سيرفر الـ API مبرمج بلغة Node.js ويستخدم بيئة Express للتعامل الفوري والآمن مع المعاملات.

**خطوات التشغيل:**
1. تأكد من تثبيت Node.js على الخادم أو جهازك الشخصي.
2. افتح مجلد `/server` في الـ Terminal وقم بتنصيب المكتبات اللازمة:
   ```bash
   npm init -y
   npm install express cors jsonwebtoken
   ```
3. قم بتشغيل السيرفر مباشرة:
   ```bash
   node api_server.js
   ```
4. لتأمين تشغيل السيرفر في الخلفية بشكل دائم على سيرفر الـ VPS، يوصى باستخدام **PM2**:
   ```bash
   npm install pm2 -g
   pm2 start api_server.js --name "kurotek-security-api"
   pm2 save
   pm2 startup
   ```

### 3. استضافة لوحة تحكم الويب (`admin_panel.html`)
- لأن اللوحة مبنية كصفحة برميوم تفاعلية مستقلة (Single Page Application) ومزينة بـ Tailwind CSS، لا تتطلب أي سيرفر تشغيل خاص بها!
- يمكنك رفع ملف `admin_panel.html` مباشرة على أي استضافة ويب عادية أو استضافة مجانية مثل Vercel أو Netlify أو cPanel أو حتى تشغيله محلياً بمتصفحك.
- بعد فتح اللوحة، قم بتغيير رابط الـ API في الحقل المخصص أعلى الشاشة ليشير إلى سيرفر الـ VPS الخاص بك (مثل: `https://api.yourdomain.com`).

## 🔒 ثانياً: تفاصيل الهيكلية والبروتوكول الأمني الآمن (Secure API)

لضمان أعلى معايير الحماية الأمنية المتبعة في الأنظمة المصرفية، تم تطبيق البروتوكول الأمني التالي:

1. **المصادقة المشفرة بـ JWT (JSON Web Tokens)**:
   - يتم توقيع الجلسة الإدارية بلوحة التحكم الويب بـ JWT صالح لمدة 24 ساعة، ويتم التحقق منه تلقائياً في كل عملية CRUD لإضافة أو تعديل أو حذف التراخيص.
2. **منع هجمات التكرار وتزوير الهوية (Anti-Replay & Anti-Tampering)**:
   - يتبادل التطبيق مع السيرفر توقيعاً رقمياً فريداً مشفراً بـ **HMAC-SHA256** باستخدام مفتاح سري مشترك (`HMAC_SECRET`).
   - يتكون التوقيع من تشفير دمج (السيريال + معرّف الجهاز + طابع زمني دقيق + رمز Nonce عشوائي).
   - إذا حاول أحد المخترقين التقاط الطلب وتعديل السيريال أو الجهاز، سيرفض السيرفر الطلب فوراً لعدم مطابقة التوقيع.
   - إذا تم تكرار الطلب الملتقط بعد مرور أكثر من 5 دقائق، يتم رفضه أمنياً (Block Replay Attack) بناءً على فحص الطابع الزمني الملحق بالطلب.
3. **مكافحة هجمات SQL Injection**:
   - تتم فلترة وتطهير جميع المدخلات الحساسة (السيريالات والمعرفات) عبر تعبيرات برمجية منتظمة (Regex) وصارمة تسمح فقط بالحروف والأرقام والشرطات، مما يلغي تماماً إمكانية دمج أي تعليمات برمجية خبيثة.

## 📱 ثالثاً: آليات تفعيل التطبيق وتأمين الجهاز (Device-Bound Security)

### 1. معرّف الجهاز الآمن (Device-ID Locking)
عند فتح التطبيق لأول مرة، يقوم التطبيق بتوليد معرّف بصمة فريد للهاتف عبر دمج `Settings.Secure.ANDROID_ID` الموثق مع معرف UUID مشفر يتم حفظه داخل ذاكرة التطبيق الآمنة.
- يتم إرسال هذا المعرّف تلقائياً عند طلب التفعيل الأول.
- يقوم السيرفر بربط هذا السيريال بهذا الجهاز بالذات.
- في حال محاولة استخدام نفس السيريال على هاتف آخر، سيقوم السيرفر برفض الطلب لأن المعرّف المسجل لا يطابق معرّف الهاتف الجديد (Anti-Cloning).

### 2. محرك الفحص الاستباقي للبيئة (Zero-Trust Environment Checking)
يتضمن كلاس `DeviceSecurity.kt` آليتين متقدمتين لحماية بيانات ومبيعات كروت الشبكة:
- **Root Detection**: فحص كامل لوجود ملفات كسر الحماية الثنائية مثل `su` أو تطبيقات `SuperSU`/`Magisk` وإصدارات النظام المخصصة لمطوري الطرف الثالث (`test-keys`).
- **Emulator Detection**: التعرف الفوري على تشغيل التطبيق داخل بيئات المحاكاة والـ Sandbox المشهورة (مثل Genymotion و BlueStacks والـ SDK Emulator الافتراضي) لمنع كشف ملفات الكروت أو تحليل السلوك محلياً.

في حال كشف أي تهديد، تظهر بطاقة تحذير أمني مميزة بلون قرمزي داكن في شاشة التفعيل لإرشاد المستخدم النهائي بوجود تهديدات محتملة على جهازه.

## 🛠️ رابعاً: تجهيز التطبيق للإنتاج (Production Ready) ونشر حزم الـ APK و AAB

### 1. تفعيل تشفير الأكواد وتصغير الحجم (R8 / ProGuard)
تم تفعيل المعالج المحسن لـ R8 في ملف `app/build.gradle.kts` وإعداده لاستيراد ملف `proguard-rules.pro`.
لتطبيق ضغط الأكواد وحجب التوابع بالكامل لنسخة الإنتاج:
```kotlin
buildTypes {
    release {
        isMinifyEnabled = true // تشغيل المترجم المحجب والأمان
        isShrinkResources = false // يفضل تفعيله عند توافر موارد حجمية كبيرة
        proguardFiles(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro")
        signingConfig = signingConfigs.getByName("release")
    }
}
```

### 2. إعداد مفتاح التوقيع الرسمي (Signing Configuration)
لإنتاج حزمة تفعيل APK صالحة للاختبار وتتجاوز حواجز الحماية الخاصة بـ **Google Play Protect**:
1. افتح مشروعك في **Android Studio**.
2. من القائمة العلوية اختر **Build > Generate Signed Bundle / APK**.
3. اختر **Android App Bundle** (للنشر الرسمي على المتجر) أو **APK** (للتجربة المباشرة خارج المتجر).
4. قم بإنشاء ملف Keystore مشفر جديد واحتفظ بكلمة المرور الخاصة به.
5. سيقوم الـ Android Studio بتوليد الملف النهائي وتوقيعه رقمياً بنجاح وبشكل آمن تماماً.

### 3. مطابقة سياسات متجر Google Play
- **تحديد الـ Target SDK**: يدعم التطبيق أحدث إصدار مستقر (Target SDK 34) ومتوافق بالكامل مع نظام أندرويد 10 وما فوق لضمان التوافقية بنسبة 100%.
- ** Play Integrity API**: تم إعداد التطبيق ليتوافق مع خدمات سلامة المتجر للتأكد من عدم وجود تعديلات على حزمة الـ APK أو التلاعب بتصريح تفعيل التطبيق.
- **الحد الأدنى من الصلاحيات الحساسة**: التطبيق لا يطلب أي صلاحيات إضافية غير ضرورية في الخلفية ويقتصر على معالجة صلاحيات قراءة الرسائل محلياً للتأكيد وإرسال الكروت تلقائياً للمستخدمين بنقرة واحدة بموجب السياسات الرسمية للمتجر.

## 🌟 خامساً: خطوات تفعيل الموزع الجديد والعميل (CUJ)

1. يقوم العميل بتثبيت وفتح التطبيق، فتظهر له شاشة التفعيل وبها معرّف جهازه الفريد (مثال: `9774D56D682E549C`).
2. يضغط العميل على زر **نسخ 📋** لنسخ معرّف جهازه ويرسله إليك عبر الواتساب.
3. تفتح أنت **لوحة تحكم الويب الخارجية (`admin_panel.html`)**، وتسجل دخولك كمدير.
4. تملأ بيانات العميل (الاسم، اسم شبكته، رقم هاتفه، ومدة الاشتراك) وتضغط على **إنشاء وتشفير الترخيص 🔐**.
5. ستقوم اللوحة بإنشاء وتوليد السيريال الفريد تلقائياً (مثال: `771112223-KS0D8F3E`).
6. ترسل هذا السيريال للعميل. بمجرد إدخاله في هاتفه، سيقوم التطبيق بربط السيريال بمعرف جهازه بشكل دائم ويعمل التطبيق بجميع مميزاته بشكل آمن!
7. إذا تعطل هاتف العميل أو أراد الانتقال لهاتف جديد، يمكنك ببساطة الضغط على زر **إعادة ضبط الهاردوير 🔄** بجانب اسمه في لوحة التحكم لإعادة الترخيص للوضع غير المستخدم ليرتبط بالهاتف الجديد بكل سلاسة وأمان!
