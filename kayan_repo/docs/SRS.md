# Software Requirements Specification (SRS)
# منصة Kurotek — نظام إدارة كروت الشحن SaaS

**الإصدار:** 3.0 — تحديث Phase 6  
**التاريخ:** 2026-07-04  
**الحالة:** ✅ معتمد وسارٍ

---

## 1. وصف المنصة (System Overview)

Kurotek هي منصة SaaS متعددة المستأجرين (Multi-Tenant) مصممة لأصحاب محلات بيع كروت الشحن والإنترنت في اليمن. تتيح المنصة:
- **أتمتة استلام وتوزيع كروت الشحن** بناءً على رسائل SMS من المحافظ المالية اليمنية.
- **إدارة مالية شاملة للموزع**: مبيعات، أرباح، مصروفات، ذمم، تدفق نقدي.
- **لوحة تحكم SaaS** للمدير العام مع نظام اشتراكات وتفعيل.

---

## 2. المستخدمون (Actors)

| الدور | الوصف |
|---|---|
| `super_admin` | مدير المنصة الكامل: يدير الخطط، السيريالات، المستأجرين |
| `tenant_admin` | صاحب المحل: إدارة كاملة للحساب بما في ذلك تجاوز حد الائتمان |
| `operator` | موظف: تسجيل مبيعات فقط، لا يمكنه تجاوز حد الائتمان |
| `viewer` | مشاهدة التقارير فقط |

## 3. المتطلبات الوظيفية (Functional Requirements)

### FR-01: نظام المصادقة (Authentication)
- تسجيل دخول بـ JWT (Access Token 15 دقيقة + Refresh Token 7 أيام)
- تسجيل مستأجر جديد مع فترة تجريبية 7 أيام
- تفعيل الاشتراك بالسيريال + كود التفعيل

### FR-02: أتمتة SMS (SMS Engine)
- استلام رسائل SMS من المحافظ اليمنية (جيب، جوالي، كريمي، ون كاش، إم فلوس، حاسب)
- تحليل الرسائل بـ Regex مخصص لكل محفظة
- توزيع كروت الشحن تلقائياً وإرسالها بـ SMS
- إدارة الطلبات المعلقة (Pending Approvals)
- ضمان عدم تكرار المعالجة (Idempotency Key)

### FR-03: إدارة المخزون والكروت
- رفع كروت بالجملة (Bulk Upload) بالنص
- إدارة الفئات (100، 200، 250، 300، 500)
- تتبع حالة الكارت (مستخدم / متاح)

### FR-04: موديول حاسبة الموزع

#### FR-04.1: إدارة التسعيرة
- تعريف سعر الشراء وسعر البيع لكل فئة ونوع كرت (REGULAR / PRO)
- حفظ Snapshot للأسعار وقت كل عملية بيع (للتقارير التاريخية)
- إعادة الضبط للأسعار الافتراضية

#### FR-04.2: إدارة العملاء / البقالات
- إنشاء / تعديل / حذف بيانات البقالات
- تعيين حد ائتمان (Credit Limit) لكل عميل
- **قاعدة Enforcement:** منع تسجيل بيع آجل يتجاوز الحد تلقائياً
- **استثناء:** `tenant_admin` يمكنه تجاوز الحد مع إرفاق سبب
- عرض إجمالي المديونية الحالية لكل عميل
- عرض تاريخ آخر دفعة
- **تنبيه:** إرسال alert عند بلوغ 80% من الحد
- قائمة "العملاء المتعثرين" (لم يدفعوا > 30 يوم)

#### FR-04.3: تسجيل المبيعات
- تسجيل فاتورة: الفئة، الكمية، المبلغ المقبوض
- حساب تلقائي: `totalAmount = sellPrice × quantity`, `profit = (sellPrice - buyPrice) × quantity`
- تحديد تلقائي إذا كانت البيعة آجلة `isCredit = receivedAmount < totalAmount`
- ربط البيعة بعميل (اختياري للبيع النقدي)

#### FR-04.4: التحصيل والمدفوعات
- تسجيل دفعة من عميل (تقليص الذمة)
- عرض كشف حساب العميل التفصيلي

### FR-05: لوحة الأرباح الاحترافية (Profit Board)

| المؤشر | الوصف |
|---|---|
| أرباح اليوم | مجموع `profit` لعمليات اليوم |
| أرباح الأسبوع | مجموع `profit` آخر 7 أيام |
| أرباح الشهر | مجموع `profit` الشهر الحالي |
| أرباح السنة | مجموع `profit` السنة الحالية |
| أرباح بين تاريخين | مجموع `profit` حسب نطاق تاريخ مخصص |
| صافي الربح | `grossProfit - totalExpenses` |
| إجمالي المبيعات | مجموع `totalAmount` |
| تكلفة البضاعة | `buyPrice × quantity` |
| هامش الربح % | `(grossProfit / totalSales) × 100` |
| متوسط الربح اليومي | `monthProfit / daysInMonth` |
| أفضل الأصناف ربحاً | تجميع حسب `categoryValue` + `cardType` |
| أفضل العملاء | ترتيب حسب إجمالي الشراء |
| أكثر الفئات مبيعاً | تجميع حسب `categoryValue` |

### FR-06: نظام التدفق النقدي (Cash Flow)
- **الرصيد الافتتاحي:** مجموع injections - withdrawals قبل الفترة
- **المقبوضات:** مبيعات نقدية + تحصيل ذمم + injections رأس مال
- **المصروفات:** مبالغ Expenses + withdrawals رأس مال
- **صافي الحركة:** مقبوضات - مصروفات
- **الرصيد الختامي:** الافتتاحي + صافي الحركة
- بيانات رسم بياني يومي (Sparkline data)

### FR-07: نظام المصروفات
- تصنيفات ثابتة: إيجار، رواتب، كهرباء، إنترنت، نقل، صيانة، أخرى
- تصنيفات مخصصة: يمكن للمستخدم إضافة تصنيف جديد (`ExpenseCategory` model)
- تصفية المصروفات حسب التصنيف + التاريخ
- إجمالي المصروفات حسب الفئة (Breakdown Chart)

### FR-08: لوحة التحكم الرئيسية (Main Dashboard)
- بطاقات: مبيعات اليوم، أرباح اليوم، الذمم الإجمالية، المخزون المتاح، الطلبات المعلقة
- رسم بياني: مبيعات آخر 30 يوم
- أعلى العملاء (Top 5)
- أعلى المنتجات ربحاً (Top 5)
- أكثر المحافظ استخداماً
- آخر 10 عمليات بيع
- **تنبيهات مُدمجة:**
  - 🔴 عملاء تجاوزوا حد الائتمان
  - 🟡 عملاء بلغوا 80% من الحد
  - 🟠 مخزون منخفض (< 10 كروت لأي فئة)
  - 🔵 اشتراك ينتهي خلال 7 أيام
  - 🔴 طلبات معلقة تنتظر الموافقة

### FR-09: نظام التقارير
- فلتر حسب: التاريخ، العميل، الفئة، نوع الكرت، المحفظة
- تصدير PDF (مكتبة pdfmake أو puppeteer)
- تصدير Excel (مكتبة xlsx / exceljs)
- دعم الطباعة (Print-friendly endpoint)

## 4. المتطلبات غير الوظيفية (Non-Functional Requirements)

- **Multi-Tenant Isolation:** كل query تحتوي على فلتر `tenantId`
- **SOLID + DRY:** Business Logic في Services فقط، لا تكرار في Controllers
- **Prisma Transactions:** عمليات متعددة الكتابة تُنفَّذ داخل `$transaction`
- **Swagger:** جميع Endpoints موثقة بـ `@ApiTags`, `@ApiOperation`, `@ApiResponse`, `@ApiBearerAuth`
- **Validation:** جميع DTOs تستخدم `class-validator` + `ValidationPipe`
- **Performance:** Indexes مناسبة على حقول الفلترة والبحث الشائعة
- **Unit Tests:** Coverage للخدمات الحساسة (حساب الأرباح، Credit Limit، Cash Flow)
